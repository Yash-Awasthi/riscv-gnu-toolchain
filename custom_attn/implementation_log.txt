 OVERVIEW
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Custom RISC-V instruction: attn rd, rs1, rs2
  Mathematical operation:    Attention(Q, K, V) = softmax(Q*K^T / sqrt(d_k)) * V

  Two structs:
    1. attn_dims_t : { rows, cols, seq_len, d_model }
    2. attn_qkv_t  : { float *Q, float *K, float *V }

  The C program calls __builtin_riscv_attn() — NO inline assembly.
  The modified GCC compiler emits the "attn" instruction automatically.

 [STEP 13]  GIMPLE Idiom Detection Pass — File 7
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Beyond the explicit builtin (Steps 4-8), we implemented a GIMPLE
  optimization pass that automatically detects the 4-stage attention
  pattern in plain C loops and replaces it with __builtin_riscv_attn().

  The pass (riscv_attn_detect) runs after GCC's loop pass, where:
    - Loops are in canonical form (single entry, single exit)
    - SCEV (scalar evolution) is available for trip count analysis
    - Array accesses are still recognizable as ARRAY_REF trees

  It detects 4 consecutive top-level loops matching:
    Stage 1: score[i][j] += Q[i][k] * K[j][k]   (matmul Q × K^T)
    Stage 2: score[i][j] /= sqrt(d_model)         (scaling)
    Stage 3: softmax per row (max, exp-sum, normalize)
    Stage 4: output[i][j] += score[i][k] * V[k][j] (matmul score × V)

  Source file: custom_attn/src/riscv-attn-detect.cc (~1169 lines)
